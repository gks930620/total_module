# 앱 요청과 백엔드 수신 부분코드 정리

## 목적
- 앱에서 **어떤 코드로 API를 요청하는지(부분 코드만)**,
- 백엔드에서 **어떤 코드로 요청을 받는지(부분 코드만)** 를 빠르게 확인하기 위한 문서.

---

## 1) 기본 주소 규칙

### 앱 기준 Base URL 구성
파일: `businesscard_qr_app/lib/api_config.dart`

```dart
if (Platform.isAndroid) {
  return 'http://10.0.2.2:8080';
}
return 'http://localhost:8080';
```

### 백엔드 URL 규칙
- 컨텍스트패스 없이, 컨트롤러에서 직접 `/api/...` 로 시작.
- 실제 API 형태: `.../api/...`

## 2) 앱에서 요청하는 핵심 부분 코드

파일: `businesscard_qr_app/lib/services/business_card_service.dart`

### (a) 로그인 직후 사용자 동기화 요청
```dart
Future<void> syncUser(kakao.User kakaoUser) async {
  final userId = 'kakao_${kakaoUser.id}';
  final response = await http.post(
    ApiConfig.uri('/api/users/sync'),
    headers: _headers(),
    body: jsonEncode({
      'id': userId,
      'email': kakaoUser.kakaoAccount?.email,
      'nickname': kakaoUser.kakaoAccount?.profile?.nickname,
      'provider': 'kakao',
    }),
  );
  _extractData(response);
}
```

### (b) 명함 목록 조회 요청
```dart
final response = await http.get(
  ApiConfig.uri('/api/business-cards'),
  headers: _headers(userId: userId, includeJsonContentType: false),
);
final data = _extractData(response);
```

### (c) 명함 저장 요청 (multipart: payload + 이미지)
```dart
final request = http.MultipartRequest(
  isUpdate ? 'PUT' : 'POST',
  ApiConfig.uri(path),
);
request.headers.addAll(_headers(userId: userId, includeJsonContentType: false));
request.fields['payload'] = jsonEncode(payload);
if (businessCardImage != null) {
  request.files.add(await http.MultipartFile.fromPath(
    'businessCardImage',
    businessCardImage.path,
  ));
}
```

### (d) 공통 헤더: 사용자 식별 전달
```dart
if (userId != null && userId.isNotEmpty) {
  headers['X-User-Id'] = userId;
}
```

### (e) 공통 응답 파싱 (ApiResponse 형식)
```dart
if (response.statusCode < 200 || response.statusCode >= 300) {
  throw Exception(body['message']?.toString() ?? '요청 실패');
}
if (body['success'] != true) {
  throw Exception(body['message']?.toString() ?? '요청 실패');
}
return body['data'];
```

## 3) 백엔드에서 요청받는 핵심 부분 코드

### (a) 사용자 동기화 수신
파일: `src/main/java/com/test/test/user/controller/UserController.java`

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @PostMapping("/sync")
    public ResponseEntity<ApiResponse<UserResponse>> syncUser(
            @Valid @RequestBody UserSyncRequest request
    ) {
        UserEntity user = userService.syncUser(request);
        return ResponseEntity.ok(ApiResponse.success("사용자 동기화 성공", UserResponse.from(user)));
    }
}
```

### (b) 명함 API 수신 (헤더 + multipart)
파일: `src/main/java/com/test/test/businesscard/controller/BusinessCardController.java`

```java
@RestController
@RequestMapping("/api/business-cards")
public class BusinessCardController {

    @GetMapping
    public ResponseEntity<ApiResponse<List<BusinessCardResponse>>> getBusinessCards(
            @RequestHeader("X-User-Id") String userId
    ) { ... }

    @PostMapping(consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    public ResponseEntity<ApiResponse<BusinessCardIdResponse>> createBusinessCard(
            @RequestHeader("X-User-Id") String userId,
            @RequestPart("payload") String payload,
            @RequestPart(value = "businessCardImage", required = false) MultipartFile businessCardImage
    ) { ... }

    @PutMapping(value = "/{cardId}", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    public ResponseEntity<ApiResponse<BusinessCardIdResponse>> updateBusinessCard(
            @RequestHeader("X-User-Id") String userId,
            @PathVariable String cardId,
            @RequestPart("payload") String payload,
            @RequestPart(value = "businessCardImage", required = false) MultipartFile businessCardImage
    ) { ... }
}
```

### (c) 서비스 계층에서 payload 파싱/검증 후 저장
파일: `src/main/java/com/test/test/businesscard/service/BusinessCardService.java`

```java
private BusinessCardUpsertRequest parseAndValidatePayload(String payload) {
    if (payload == null || payload.isBlank()) {
        throw new BusinessRuleException("payload는 필수입니다.");
    }
    BusinessCardUpsertRequest request = objectMapper.readValue(payload, BusinessCardUpsertRequest.class);
    Set<ConstraintViolation<BusinessCardUpsertRequest>> violations = validator.validate(request);
    if (!violations.isEmpty()) {
        throw new BusinessRuleException(violations.iterator().next().getMessage());
    }
    return request;
}
```

```java
public BusinessCardIdResponse createBusinessCard(String userId, String payload, MultipartFile businessCardImage) {
    validateUserId(userId);
    BusinessCardUpsertRequest request = parseAndValidatePayload(payload);
    String imagePath = storeImage(cardId, businessCardImage);
    BusinessCardEntity entity = request.toEntity(cardId, userId, imagePath);
    businessCardRepository.save(entity);
    return new BusinessCardIdResponse(entity.getId());
}
```

## 4) 앱 함수 ↔ 백엔드 엔드포인트 매핑

| 앱 함수(요청) | HTTP | 백엔드 수신 |
|---|---|---|
| `syncUser()` | `POST /api/users/sync` | `UserController.syncUser` |
| `getAllBusinessCards()` | `GET /api/business-cards` | `BusinessCardController.getBusinessCards` |
| `getBusinessCard(id)` | `GET /api/business-cards/{cardId}` | `BusinessCardController.getBusinessCard` |
| `saveBusinessCard()` (신규) | `POST /api/business-cards` | `BusinessCardController.createBusinessCard` |
| `saveBusinessCard()` (수정) | `PUT /api/business-cards/{cardId}` | `BusinessCardController.updateBusinessCard` |
| `deleteBusinessCard(id)` | `DELETE /api/business-cards/{cardId}` | `BusinessCardController.deleteBusinessCard` |
| `incrementViewCount(id)` | `POST /api/business-cards/{cardId}/view-count` | `BusinessCardController.incrementViewCount` |
| `generateVcfDownloadToken(cardId)` | `GET /api/business-cards/{cardId}/vcf-download-url` | `BusinessCardController.getVcfDownloadUrl` |
| `generateImageDownloadToken(cardId)` | `GET /api/business-cards/{cardId}/image-download-url` | `BusinessCardController.getImageDownloadUrl` |

## 5) 확인 포인트
- 앱에서 인증 토큰이 아니라 현재는 `X-User-Id` 헤더로 사용자 식별.
- 성공 응답은 `ApiResponse(success=true, message, data)` 규격으로 처리.
- 명함 저장/수정은 `multipart/form-data`로 `payload(JSON 문자열)` + `businessCardImage` 조합.
