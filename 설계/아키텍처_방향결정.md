## 1. 확정된 방향 (한 장 요약)

```
GitHub 저장소 1개 (total_module)  =  Railway 프로젝트 1개
빌드: 루트 단일 Dockerfile → 이미지 1개 (모든 모듈 jar 포함)
실행: 모듈 1개 = Railway 서비스 1개 (APP_MODULE로 역할 선택)
접근: 각 서비스가 자기 공개 URL로 직접 트래픽 받음 — ⚠️ 게이트웨이 없음
```

| 항목 | 결정 |
|------|------|
| 저장소/Railway 프로젝트 | **각 1개 고정.** 모듈이 늘어도 서비스만 추가 |
| 실행 단위 | **모듈 = 서비스 (1:1).** jar 하나로 합치지 않는다 |
| 외부 접근 | **각 서비스의 자기 공개 URL 직행.** 프록시/게이트웨이 안 둔다 |
| dist_api_gateway | **은퇴 (모듈 삭제 예정).** 아래 §4 전환 계획 참고 |
| 새 프로젝트 편입 | 로컬에서 완성 → 폴더 투하 → 등록 절차 + env 몇 개 (§3) |

## 2. 왜 이렇게 결정했나

### 2-1. 왜 "모듈 = 서비스"인가 (jar 하나로 안 합치는 이유)
- 이 저장소의 컨셉은 **"로컬에서 완성한 프로젝트를 그대로 가져와 배포"**.
- jar 하나로 합치는 진짜 모놀리스는 가져올 때마다 **병합 수술**이 필요하다
  (SecurityConfig/application.yml/빈 이름/의존성 버전 충돌 + 한 프로젝트 장애가 전체 전파 + 전체 재배포).
- 모듈=서비스면 가져오는 비용이 **등록 절차 + 환경설정 몇 개로 고정**되고, 프로젝트끼리 충돌할 방법이 없다.

### 2-2. 왜 게이트웨이를 없애나
- 게이트웨이는 **초기 설계(서버 1대로 합쳐 배포)의 유산**. 그 세계에서는 유일한 현관문이라 필수였다.
- 실제 구현이 "서비스 분리"로 바뀐 순간, 각 서비스가 자기 공개 URL을 가지므로 존재 이유가 사라짐.
- 현재 게이트웨이는 중앙 인증/로깅/제한 등 게이트웨이다운 일을 **하나도 하지 않고** 단순 중계만 한다.
- 남는 것은 비용뿐: 서버 1대 추가, 장애 지점 추가, 프록시 특유의 복잡성(스트리밍/리다이렉트/타임아웃), 새 프로젝트마다 라우팅·프리픽스 공사.
- 요청량이 적은 사이드 프로젝트 모음이라 "도메인 단일화" 가치도 없음. 필요해지면 그때 붙이는 게 쉽다(역방향은 어려움).

### 2-3. 왜 Railway 프로젝트는 1개인가
- 같은 프로젝트 안의 서비스끼리 내부 네트워크·환경변수 참조(`${{mysql.MYSQLHOST}}`) 가능, 대시보드 일원화.
- 새 모듈 = **서비스 추가**(+ Add)이지, Railway 프로젝트 추가가 아니다.

## 3. 새 프로젝트(A, B, ...) 편입 표준 절차

> 전제: 프로젝트는 **로컬에서 완성**해서 가져온다. 여기서는 배포 적합화만 한다.

1. 프로젝트 폴더를 total_module 루트에 투하
2. `settings.gradle`에 `include 'project_a'`
3. 루트 `Dockerfile`의 COPY / bootJar 대상에 추가
4. `start.sh`(Dockerfile 내장)의 APP_MODULE 허용 목록에 추가
5. 코드 확인 (배포 적합화 최소 세트):
   - DB 등 설정이 env 주입인지 (`${SPRING_DATASOURCE_URL:로컬기본값}` 패턴)
   - `server.port: ${PORT:...}`
   - `GET /healthz`
   - 파일 저장이 로컬 디스크가 아닌지 / 비밀값이 소스에 없는지
6. Railway `+ Add` → 같은 저장소로 서비스 생성 → `APP_MODULE=project_a` + env → Generate Domain
7. 클라이언트(앱/웹)는 그 서비스의 공개 도메인을 직접 바라봄

- ~~게이트웨이에 라우팅 추가~~ ← **없음. 이 단계는 존재하지 않는다.**
- 화면+API 일체형 스프링부트면 통째로 넣으면 되고 CORS도 불필요.
  화면이 별도 정적 호스팅(CSR 분리형)이면 CORS(해당 오리진) + 인증방식(세션이면 주의)만 점검.
  상세: [`웹프로젝트_통합_설계기준.md`](웹프로젝트_통합_설계기준.md)
