1) 프로젝트 구조
/My-Total-Project (상위 폴더 - 바구니)
  ├── .gitignore             # 1. 전체 프로젝트용 깃 방어막
  ├── .dockerignore          # 2. 도커 빌드용 방어막
  ├── Dockerfile             # 3. 전체 조립 설계도 (지휘관)
  ├── settings.gradle        # 4. 하위 모듈 명단 (include 'side-spring' 등)
  ├── build.gradle           # 5. 하위 모듈 공통 규칙 (Java 버전 등)
  │
  ├── /side-spring          # [모듈1 ] 백엔드 (Spring Boot)
  ├── /side-node             # [모듈 2] 프론트엔드 (Node/React/Vue 등)
  ├── /side-static           # [모듈 3] 단순 HTML/CSS 정적 사이트
  └── /dist_api_gateway          # [모듈 0] ⭐API 게이트웨이 (배포용 Spring)
        ├── src/main/java... # 경로 안내용 코드 (매우 짧음)
        └── build.gradle     # 1, 2, 3번을 내 몸안으로 합치는 설정


이 통합 프로젝트에 사이드 프로젝트를 추가할 때
1. 사이드 프로젝트는 독립적으로 개발한다 (각자 build.gradle, src 등)
2.  통합프로젝트( 인텔리제이에서  루트폴더)에서 module 추가 -> 추가한 사이드프로젝트 선택
3.  spring 프로젝트일 때는  import module from external model (Gradle) 선택
    아닐 때는  Create module from existing sources 선택

    
Merge PR 
- 사이드 프로젝트마다 해당 사이드프로젝트의 브랜치를 만들고 작업한다
- 작업이 끝나면 PR을 만들어서 리뷰 받고, 머지한다
