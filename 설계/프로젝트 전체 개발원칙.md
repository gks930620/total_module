# 프로젝트 전체 개발원칙

이 서버는 **백엔드 API 서버**다.  
새 프로젝트 붙일 때 기본 작업은 **API + Security + DB매핑 + CORS**만 한다.

## 새 프로젝트 추가할 때 실제로 할 것

1. 패키지 만든다  
- `com.test.test.{projectName}`

2. API 만든다  
- Controller/Service/Repository/Entity 작성
- URL은 전부 `/{projectName}/api/...` 로 시작

3. Security 적용한다 (가장 단순한 방법)  
- 기존 프로젝트 Security 코드 복붙해서 `{projectName}` 버전 만든다
- 아래 URL 문자열만 전부 새 프로젝트명으로 바꾼다
  - `/project1/api/login` -> `/{projectName}/api/login`
  - `/project1/api/logout` -> `/{projectName}/api/logout`
  - `/project1/api/...` -> `/{projectName}/api/...`
- 로그인 필터 URL도 바꾼다
  - `setFilterProcessesUrl("/{projectName}/api/login")`
- 쿠키 path도 바꾼다
  - `/{projectName}`

4. DB 매핑 추가한다  
- `application-local.yml`, `application-prod.yml`에만 추가
- `app.datasource.projects.{projectName}={dbName}`
- 예: `app.datasource.projects.project2=project2_db`

5. CORS 추가한다  
- 새 프로젝트 프론트 정적 서버 도메인 허용 목록에 추가

6. (OAuth 쓰면) redirect URI만 프로젝트 경로로 맞춘다  
- `/{projectName}/login/oauth2/code/...`

## 하지 말 것

- 기본적으로 `application-projectN.yml` 만들지 않는다.
- 공통 패키지(`com.test.test`)에 프로젝트 전용 코드 넣지 않는다.
- API 경로를 프로젝트 prefix 없이 만들지 않는다.
