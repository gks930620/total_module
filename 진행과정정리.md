##  배포 후 오류 수정






### 2026-04-29 - Railway + app save(500) root cause fix
- Symptom from app: `BusinessCardService.saveBusinessCard` returned `Request failed (500)` while creating a card with image.
- Checked deployed endpoints:
  - `https://distapigateway-production.up.railway.app/` -> gateway is UP and target URL is set to business service URL.
  - `https://businesscardqr-production.up.railway.app/healthz` -> business service is UP.
- Reproduced with direct HTTP calls:
  - `POST multipart` to gateway `/api/business-cards` (without auth) returned `500`.
  - Same `POST multipart` directly to business service returned expected `401`.
- Conclusion:
  - Railway base URL was not the root cause.
  - Gateway had a multipart proxy issue before upstream forwarding.
- Fix applied:
  - File: `dist_api_gateway/src/main/resources/application.yml`
  - Added `spring.servlet.multipart.enabled: false`
  - Reason: gateway must forward multipart body as raw bytes; multipart resolver must not parse request first.
- Build verification:
  - `./gradlew --no-daemon :dist_api_gateway:bootJar` success.
