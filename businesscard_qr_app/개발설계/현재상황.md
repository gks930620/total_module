## 1) 이 프로젝트가 하는 일

Flutter 모바일 앱(명함 QR)입니다.  
카카오 로그인 후 본인 명함 데이터를 Supabase에 저장/조회하고, 명함 정보를 vCard(VCF)와 명함 이미지로 공유할 수 있게 QR 코드를 생성합니다.

핵심 플로우:
1. 앱 시작 시 `.env` 로드, Kakao SDK/Supabase 초기화
2. 카카오 토큰 자동 로그인 체크
3. 명함 목록 조회
4. 명함 등록/수정 (이미지 업로드 + VCF 생성/업로드)
5. 상세 화면에서 VCF/이미지 다운로드용 QR 표시 (Signed URL 5분)

## 2) 실제 구현 상태 (완성된 동작)

### 인증/세션
- 카카오 로그인(카카오톡 앱/카카오 계정 분기) 구현됨
- 앱 시작 시 토큰 존재 확인 후 자동 로그인 구현됨
- 로그아웃 구현됨
- 로그인 사용자 정보를 Supabase `users` 테이블에 upsert 시도

### 명함 CRUD
- 목록 조회 구현됨 (`is_active=true`, 사용자별 필터, 최신순)
- 등록 구현됨
  - 필수값 검증(이름, 전화번호)
  - 선택값 입력 필드(이메일/웹사이트/조직/직함/주소/메모 등)
  - 이미지 선택(카메라/갤러리)
  - 이미지 업로드 + vCard 생성 + DB upsert
- 수정 구현됨
  - 기존 값 로드 후 수정
  - 이미지 교체 시 기존 이미지 정리 시도
- 상세 조회 구현됨
  - 섹션형 정보 표시
  - VCF QR 생성
  - 이미지가 있으면 이미지 QR 생성

### QR/파일
- VCF 파일 생성 유틸(`vCard 3.0`) 구현됨
- Supabase Storage 업로드 구현됨
- Signed URL(5분) 생성 후 QR로 노출 구현됨

### 이미지
- 이미지 URL 기반 로컬 캐시(`ImageCacheService`) 구현됨
- 상세 화면에서 캐시 우선 표시, 없으면 네트워크 로드

## 3) 코드 기준으로 확인된 미완성/불일치

### A. 사용자 삭제 기능은 백엔드 메서드만 있고 UI 미구현
- `deleteBusinessCard()`는 있으나 화면에서 호출되지 않음
- 현재 사용자 입장에서는 명함 삭제 불가

### B. DB 스키마와 앱 코드 불일치 가능성 큼
- 앱 코드는 `business_cards.user_id`를 전제로 조회/저장
- 그런데 `migrations/create_business_cards_table.sql`에는 `user_id` 컬럼 정의가 없음
- 실제 운영 DB를 수동 변경해 쓰는 상태가 아니면, 쿼리 실패 가능성이 높음

### C. Storage 버킷 명 불일치
- 앱 코드 버킷: `business-card`
- `setup_storage.sql` 버킷: `business-card-files`
- `make_bucket_private.sql`는 다시 `business-card` 기준
- 마이그레이션 일괄 적용 시 환경마다 스토리지 동작이 깨질 수 있음

### D. 중복/충돌 가능 마이그레이션
- `create_business_cards_table.sql`에 이미 `business_card_image_url` 있음
- `add_url_columns.sql`, `add_business_card_image_url.sql`에서 같은 컬럼 재추가 시도
- 순서/실행 상태에 따라 migration 실패 가능

### E. 구현되었지만 실제로 안 쓰는 코드 존재
- `main.dart` 내 `_BusinessCardListScreenState`의 `_loginWithKakao`, `_saveUserToSupabase`는 미사용
- `BusinessCardService`의 `_ensureBucketExists`, `_generateSimpleHash`, `incrementViewCount`도 미사용
- `download_tokens` 테이블/함수 마이그레이션이 있으나 앱에서 사용하지 않음

### F. iOS 설정 미완성
- `ios/Runner/Info.plist`에 `NSCameraUsageDescription`, `NSPhotoLibraryUsageDescription` 없음
  - 이미지 촬영/선택 기능에서 권한 이슈 발생 가능
- 카카오 로그인용 URL scheme/queries 관련 설정도 확인되지 않음

### G. 보안/운영 리스크
- 루트 `.env`에 실제 Supabase URL/anon key, Kakao key가 포함된 상태
- 저장소 관리 정책에 따라 비밀값 노출 리스크 존재
