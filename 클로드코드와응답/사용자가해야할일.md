## 1. 🔴 지금 바로 (다음 배포 전에 필수)

### ① Railway — businesscard_qr 서비스 환경변수 확인/추가

| 환경변수 | 값 | 왜 |
|----------|-----|-----|
| `SPRING_DATASOURCE_URL` | `jdbc:mariadb://<host>:<port>/<db>` (Railway DB 플러그인 값) | ⚠️ **이번 수정으로, 없으면 부팅 자체가 실패함** (전엔 조용히 H2로 떠서 재배포마다 데이터 소실). 이미 설정돼 있다면 그대로 두면 됨 |
| `SPRING_DATASOURCE_USERNAME` / `SPRING_DATASOURCE_PASSWORD` | (DB 플러그인 값) | 위와 동일 |
| `APP_JWT_SECRET` | 랜덤 긴 문자열 (기존 값 유지) | 기본값이면 배포 차단됨 (기존과 동일) |
| **`APP_PUBLIC_BASE_URL`** ← 신설 | `https://<게이트웨이 공개 도메인>` (예: `https://distapigateway-production.up.railway.app`) | **이번에 새로 생긴 변수.** QR·이미지·다운로드 URL이 이 도메인으로 생성됨. 안 넣으면 기존 X-Forwarded 방식으로 동작하지만 WARN 로그가 뜨고 확실하지 않음 |

### ② Railway — dist_api_gateway 서비스 (변경 없음, 확인만)

| 환경변수 | 값 |
|----------|-----|

| `APP_GATEWAY_BUSINESS_QR_URL` | businesscard_qr 서비스 URL (localhost면 배포 차단) |

## 2. 🟡 선택 (원하면 켜는 것)

| 환경변수 | 값 | 효과 |
|----------|-----|------|
| `APP_KAKAO_EXPECTED_APP_ID` | 카카오 개발자 콘솔의 앱 ID (숫자) | 다른 카카오 앱의 토큰으로 로그인하는 것 차단. **미설정이면 기존과 완전히 동일** |
| `APP_JWT_REFRESH_TOKEN_VALIDITY_SECONDS` | 초 단위 (기본 1209600 = 14일) | 리프레시 토큰 유효기간 조정 |

## 3. 📱 앱(Flutter) 재빌드

이번에 앱 코드도 바뀌었으므로 (타임아웃·401 복구·업로드 Content-Type 등) **APK를 다시 빌드해서 설치**해야 효과가 적용됨.

```
cd businesscard_qr_app
flutter pub get        # http_parser 의존성 추가됨
flutter build apk      # 또는 flutter run
```

- `.env`의 `GATEWAY_BASE_URL`은 이미 Railway 게이트웨이로 설정돼 있음 — 그대로 두면 됨.
- 참고: 앱 폴더가 최상위 `businesscard_qr_app/`로 이동됐으므로 IDE에서 새 경로로 열 것.

## 4. ✅ 배포 후 눈으로 확인 (5분 체크리스트)

1. [ ] **부팅 성공?** — Railway 로그에서 businesscard_qr가 정상 기동했는지. (env 빠졌으면 이제 명확한 한글 에러와 함께 죽음 — 그게 정상 동작)
2. [ ] **QR/이미지 URL 도메인** — 앱에서 명함 만들고, QR·이미지 URL이 `https://<게이트웨이 도메인>`으로 시작하는지 실기기로 확인
3. [ ] **QR 실제 스캔** — 다른 폰 카메라로 QR 찍어서 vCard 다운로드 되는지
4. [ ] **재배포 후 데이터 보존** — 한 번 재배포하고 기존 명함·이미지가 남아 있는지
5. [ ] **콜드스타트** — 한동안(30분+) 방치 후 앱 첫 실행 시 무한 로딩 대신 "서버가 깨어나는 중" 안내 후 재시도로 성공하는지
6. [ ] **H2 콘솔 닫힘** — `https://<businesscard_qr 도메인>/h2-console` 접속 안 되는지
