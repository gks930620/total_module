# 로컬 프로젝트를 total_module로 편입할 때 정리할 것

> 작성일: 2026-07-08 (doll_gacha 편입에서 실제 겪은 것 기반)
> 등록 절차(settings.gradle/Dockerfile/Railway)는 [`아키텍처_방향결정.md`](아키텍처_방향결정.md) §3 — 이 문서는 그 **앞 단계인 "정리"** 체크리스트다.

---

## 0. 전제

편입 = **git 관리 주체가 total_module로 넘어오는 것.** 원래 프로젝트의 저장소는 "이관 전 히스토리 보관소"로만 남는다.

---

## 1. git 정리 (가장 먼저, 순서 중요)

- [ ] **내부 `.git` 삭제 전 확인**: 원 저장소에 push 안 된 커밋이 있는지
  ```bash
  cd <프로젝트폴더> && git log --oneline @{u}..HEAD | wc -l   # 0이어야 안전
  ```
  push 안 된 게 있으면 먼저 원 저장소에 push해서 히스토리 보존.
- [ ] **내부 `.git` 폴더 삭제.** 안 지우면 total_module에 **gitlink(중첩 저장소)로 커밋**되어
  clone 받을 때 내용이 통째로 빠진다 (doll_gacha에서 실제 발생 → 수습함).
- [ ] 프로젝트의 `.gitignore`는 그대로 둔다 — 중첩 .gitignore도 정상 동작하며, build/·node_modules 등을 계속 막아준다.

## 2. 파일 정리

- [ ] **클라이언트 앱이 섞여 있으면 분리**: 앱/웹 클라이언트 폴더는 total_module 최상위의 별도 폴더로
  (Gradle include 대상 아님 — businesscard_qr_app 사례). 필요 없으면 삭제.
- [ ] **빌드 산출물/로컬 데이터 확인**: `build/`, `out/`, `logs/`, `data/`(로컬 DB 볼륨), `node_modules/`,
  `dist/` — gitignore로 막혀 있는지 확인 (대개 원 프로젝트 gitignore가 이미 처리).
- [ ] **로컬 전용 파일**: `local.properties`, `.idea/` 등은 커밋 금지 (루트 .gitignore가 local.properties는 이미 막음).
- [ ] **죽은 리소스 제거**: 안 쓰는 templates/, data-*.sql, 옛 프로파일 yml 등 (dist_api_gateway에서 겪은 혼란 방지).
- [ ] **비밀값 점검**: `.env`·yml에 커밋된 시크릿 있는지. 운영 값은 전부 Railway env로.

## 3. 로컬 전용 인프라 파일 처리

- [ ] 프로젝트 자체의 `Dockerfile` / `docker-compose.yml`: **Railway는 루트 Dockerfile만 사용**하므로 배포에는 관여 안 함.
  로컬 개발용으로 남겨도 되지만, 파일 상단에 `# 로컬 개발 전용 — Railway 배포는 루트 Dockerfile` 주석을 달아 혼동 방지.

## 4. 빌드 특이사항 점검 ⚠️

- [ ] **프론트엔드 빌드(npm/vite 등)가 gradle 태스크에 묶여 있는지** 확인
  (`processResources dependsOn copyFrontend` 같은 것). 루트 Docker 빌드 환경은 **JDK만 있고 node가 없어서**
  그대로면 이미지 빌드가 깨진다. 대응: 빌드 산출물을 `static/`에 커밋해두고 npm 태스크는
  "npm이 있을 때만"(onlyIf) 돌게 가드 — 로컬에선 자동 빌드, Docker에선 커밋된 산출물 사용. (doll_gacha 사례)
- [ ] 그 외 로컬 도구 의존 태스크(파이썬 스크립트, OS 명령 등)가 빌드 경로에 있는지.

## 5. 배포 적합화 (등록 전 마지막)

[`아키텍처_방향결정.md`](아키텍처_방향결정.md) §3-5와 동일:
- [ ] DB 등 설정 env 주입 (`${SPRING_DATASOURCE_URL:로컬기본}` 패턴 — Railway `MYSQL*` 변수 호환이면 더 좋음)
- [ ] `server.port: ${PORT:...}` / **`GET /healthz`** 추가
- [ ] 파일 저장이 컨테이너 디스크가 아닌지 (DB LONGBLOB / Supabase / S3)
- [ ] 프로파일: Railway에서 어떤 프로파일로 뜨는지 명확히 (`SPRING_PROFILES_ACTIVE`)
- [ ] 웹이면 CORS·쿠키(secure/SameSite), 인증 방식 확인
